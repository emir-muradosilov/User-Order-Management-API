В проекте реализована собственная система аутентификации и авторизации на основе ролей и permissions.
Доступ к ресурсам контролируется через RBAC, реализованный на уровне БД.


Таблица roles - тут хранятся возможные роли (т.к. могут быть пользователи с разным уровнем доступа):
- id (int) — PK 
- name (string) — имя роли
-description (str) - краткое описание
-is_active (bool) - статус. Можно отозвать права сразу у всех кто относится к этой роли

Таблица permissions - указание прав доступа к действия над объектом и самим объектом:
- role_id (int) — PK 
- name (str) - права на доступ состоит из - действия и объекта действия (над которым оно будет проходить)
-description - краткое описание
-is_active - активное или выключено

Таблица role_permissions - соотношение id ролей к id правам на доступ many to many:
- role_id - id роли
- permission_id - id прав


Таблица users - список пользователей
- id (int) — PK 
- Login - логин
- email - почта (нужная функция для например сброса/изменения/восстановления пароля, для бизнес задач - рассылки и пр.)
- password_hash - т.к. пароль хранить не безопасно, мы храним хэш пароля
- is_active - статус пользователя
-create_at - дата создания / регистрации
-role_id - id роли к которой относится данный пользователь


*прав доступа (401 / 403) на mock-эндпоинте
